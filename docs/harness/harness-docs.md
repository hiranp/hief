# Harness Specification: docs

**Intent:** hief-docs
**Type:** integration

## Purpose
Validate the `hief docs` command group. The harness ensures `init` scaffolds directories and templates, `generate` produces files with resolved variables, `list` returns available templates, and `check` reports a healthy docs structure. Idempotency and offline operation are also verified.

## Environment Requirements
- [ ] In-memory database (no filesystem side effects)
- [ ] `hief.toml` with default `[docs]` configuration present in temp dir
- [ ] No `.hief/` directory pre-existing in temp dir (fresh state)

## Scenario: Fresh `hief docs` invocation

### Setup
```rust
// Code to initialize the harness environment
let db = Database::open_memory().await?;
let tmp = tempfile::tempdir()?;

// create minimal config and Cargo.toml for project detection
std::fs::write(tmp.path().join("hief.toml"), "[docs]\n")?;
std::fs::write(tmp.path().join("Cargo.toml"), "[package]\nname = \"my-project\"\n")?;

let config = Config::load(&tmp.path().join("hief.toml"))?;
let root = tmp.path();
```

### Execution Steps (LATS Trajectory)

| Step | Action | Expected State | Validation |
|------|--------|---------------|------------|
| 1 | Run `hief docs init` in the temp root | Directories `docs/specs`, `docs/harness`, `.hief/templates` created | Assert each path exists and report indicates creation |
| 2 | Execute `hief docs generate spec --name example` | `docs/specs/spec-example.md` created with unresolved placeholders | File exists and contains `{{feature}}` token |
| 3 | Call `hief docs list` | Returns all six template identifiers | Parse stdout and check for `spec`, `harness`, `golden` entries |
| 4 | Invoke `hief docs check` | Reports zero missing docs | `assert!(report.missing.is_empty())` |

### Success Criteria
- [ ] All steps complete without panic
- [ ] Database state matches expectations at each step
- [ ] No resource leaks (temp dirs cleaned up)
- [ ] Execution completes within 500 ms

### Failure Modes & Recovery

| Failure | Detection | Recovery |
|---------|-----------|----------|
| `docs/specs` exists but is a file | `io::Error` with `AlreadyExists` | Remove or rename the file, then rerun `init` |
| Template rendering fails due to missing var | placeholder remains in output | Treat as non-fatal; leave hooks for manual edit |

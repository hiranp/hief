## Overview

Add a `hief docs draft` command that uses an LLM prompt to help agents fill in `{{placeholder}}` variables in a generated SDD document.
This eliminates the manual editing step and reduces the time from "feature idea" to "approved spec" from hours to minutes.

## User Stories

### US-1: Agent drafts a spec automatically
**As an** AI coding agent,
**I want to** run `hief docs draft spec --name search` and receive a fully filled-in spec,
**So that** I can begin implementation without waiting for a human to populate boilerplate.

**Acceptance Criteria:**
- Given `hief docs generate spec --name search` has been run, When `hief docs draft spec --name search` is executed, Then `docs/specs/spec-search.md` has no remaining `{{placeholder}}` patterns.
- Given the command is run without an LLM API key configured, When the command runs, Then a clear error message is displayed with instructions for configuring the key.

### US-2: Human reviews and approves the draft
**As a** human reviewer,
**I want to** see a diff of all changes the LLM made to a spec,
**So that** I can quickly verify nothing violates the constitution before approving.

**Acceptance Criteria:**
- Given the draft has been generated, When the human runs `git diff docs/specs/`, Then all placeholder replacements are visible as normal git diff hunks.

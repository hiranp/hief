# HIEF Protocol

This skill defines the mandatory protocol for AI agents operating in this codebase.
Follow these steps for every session.

## 1. Orient (Every Session)
Call these tools immediately upon starting a new session to understand the project state:
- `get_project_context` -> High-level overview (files, intents, health).
- `get_conventions` -> Learn the project's machine-readable rules.
- `get_project_health` -> Check for regressions or active issues.

## 2. Search (Find Context)
Always search before proposing changes:
- `search_code`: Keyword search (FTS5).
- `structural_search`: AST pattern matching (e.g., `"$X.unwrap()"`).
- `related_files`: Find co-accessed files (Hebbian graph).

## 3. Intend (Task Coordination)
For all non-trivial changes:
- `create_intent`: Create a task in the dependency graph.
- `update_intent`: Set status to `in_progress` and assign it to yourself.

## 4. Execute (Make Changes)
- Follow conventions from `.hief/conventions.toml`.
- Use `structural_search` to self-check for anti-patterns before submitting.

## 5. Verify (After Changes)
- `run_evaluation`: Run golden set tests.
- `get_eval_scores`: Ensure no regressions.
- `update_intent`: Set status to `in_review`.

# Harness Specification: docs-scaffold

> Example HDD test harness document.
> Inspired by Tessl's skills-driven systematic testing approach.

**Intent:** hief-c3d4
**Type:** integration

## Purpose

Validate that `hief docs init` correctly scaffolds the `docs/specs/`, `docs/harness/`, and `.hief/templates/` directories, generates all expected files (including `SDD_LLM_PROMPT.md` and framework templates), and produces accurate CLI output.

## Environment Requirements
- [ ] Temporary directory (no filesystem side-effects on the actual project)
- [ ] `hief.toml` with default `[docs]` configuration present in temp dir
- [ ] No `.hief/` directory pre-existing in temp dir (fresh state)

## Scenario: Fresh `docs init` in empty project

### Setup
```rust
let tmp = tempfile::tempdir()?;
let root = tmp.path();

// Minimal hief.toml
std::fs::write(root.join("hief.toml"), "[docs]\n")?;

// Minimal Cargo.toml so project_name can be detected
std::fs::write(root.join("Cargo.toml"), "[package]\nname = \"my-project\"\n")?;

let config = Config::load(&root.join("hief.toml"))?;
```

### Execution Steps (LATS Trajectory)

| Step | Action | Expected State | Validation |
|------|--------|---------------|------------|
| 1 | Call `scaffold_docs_dirs(root, &config.docs)` | Returns `Ok(report)` | No error propagated |
| 2 | Check `report.directories_created` | Contains `docs/specs`, `docs/harness`, `.hief/templates/` | `assert!(report.directories_created.contains(...))` |
| 3 | Check `report.prompt_created` | `true` | `assert!(report.prompt_created)` |
| 4 | Read `SDD_LLM_PROMPT.md` | Contains "GitHub Spec-Kit" URL | `assert!(content.contains("spec-kit"))` |
| 5 | Check `frameworks/rust.md` exists | File exists | `assert!(root.join(".hief/templates/frameworks/rust.md").exists())` |
| 6 | Run `scaffold_docs_dirs` again (idempotency) | All already in `report.already_existed` | No duplicates created |

### Success Criteria
- [ ] All directories and files exist after step 2
- [ ] `SDD_LLM_PROMPT.md` content is non-empty and contains key URLs
- [ ] Framework templates (`rust.md`, `python.md`, `tauri.md`) all exist
- [ ] Second run is idempotent — no errors, no duplicate entries
- [ ] Execution completes within 500ms (no network calls)

### Failure Modes & Recovery

| Failure | Detection | Recovery |
|---------|-----------|----------|
| `docs/specs` dir already exists but owned by root | `io::Error` with `PermissionDenied` | Surface as `HiefError::Io`, suggest `sudo hief docs init` |
| `.hief` dir does not exist | `create_dir_all` fails silently | Pre-create `.hief/` in scaffold logic |
| Template constant updated but file content stale | Hash mismatch in idempotency check | Re-write file when content differs |

## Cleanup
```rust
tmp.close()?; // removes temp dir
```

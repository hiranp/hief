# Plan 01-02: Semantic Cache and Quality Metadata

## Objective

Add semantic cache and retrieval-quality instrumentation to reduce repeated query latency and seed active-eval feedback.

Purpose: Make semantic retrieval cheaper and measurable without sacrificing correctness.

Output: TTL-bounded cache path and per-query quality metadata emission.

## Tasks

### Task 1

Type: auto

TDD: true

Files: src/db.rs, src/index/vectors.rs, tests/test_semantic_cache.rs

Behavior:

- Query with high similarity within TTL returns cache hit.
- Expired entry forces recomputation and rewrites cache row.
- Different language filter yields separate cache key path.

Action:

Add database migration in src/db.rs for semantic cache records including query fingerprint, embedding hash, language scope, result payload hash, and expires_at. Before wiring the cache, verify that `VectorConfig.enabled` is set to `true` by default (or toggle it explicitly for this plan) and that the LanceDB table is initialized correctly on first run — add a startup verification assertion in the test setup. Wire cache read and write into src/index/vectors.rs around semantic search execution per RET-04. Keep fallback path deterministic when cache is disabled or stale. Add tests for hit, miss, and TTL expiration behavior.

Verify:

- automated: cargo test test_semantic_cache -- --nocapture

Done:

- Semantic search has TTL-bounded cache semantics with deterministic invalidation and passing tests.

### Task 2

Type: auto

TDD: true

Files: src/mcp/tools.rs, src/index/search.rs, src/index/vectors.rs, tests/test_semantic_cache.rs

Behavior:

- Semantic search response contains a groundedness-related quality field.
- Cache-hit and cache-miss paths both populate quality metadata.
- Metadata defaults safely when insufficient signals exist.

Action:

Add lightweight quality metadata emission in semantic and hybrid search response flow, including strategy and cache_used flag. Reserve a `quality_signal` field in the response schema but populate it with a placeholder value (0.0 or None) — the actual groundedness scorer is implemented in Phase 03-03 (EVAL-01 owner). Keep schema additive and backward-compatible so Phase 03-03 can fill the field without a breaking change. Validate the metadata path in tests without introducing network dependencies or LLM judge calls.

Verify:

- automated: cargo test test_semantic_cache -- --nocapture

Done:

- Semantic retrieval responses carry stable quality metadata with a reserved `quality_signal` field (placeholder value). No groundedness scoring is performed here — that is EVAL-01 territory owned by Phase 03-03.

## Threat Model

Trust boundaries:

- MCP query input to semantic cache keying: Untrusted input may poison cache identity.
- Semantic cache storage to tool output: Stale records can degrade retrieval trust.

STRIDE register:

- T-01-03, Tampering, semantic cache rows, mitigate: Include TTL, language scope, and payload hash validation before serving.
- T-01-04, Information disclosure, cached payload return, mitigate: Store only required result fields and re-apply output budget before response.
- T-01-SC, Tampering, cargo dependency changes, accept: No new package-manager install in this plan.

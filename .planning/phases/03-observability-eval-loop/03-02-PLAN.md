# Plan 03-02: MCP Instrumentation and Session Cost Tooling

## Objective

Instrument tool handlers and expose session-level telemetry summaries.

Purpose: Make observability consumable for daily development loops.
Output: Handler instrumentation plus session-summary exposure via resource or command.

## Context

## Tasks

### Task 1: Instrument MCP handlers for OBS-02 and complete PRO-03 audit trail

Files: src/mcp/tools.rs, tests/test_tool_events.rs

Type: auto

TDD: true

Behavior:

- Successful tool calls persist telemetry rows.
- Failed tool calls still record minimal failure telemetry.
- Instrumentation errors do not crash primary handler.

Action:

- Add scoped instrumentation wrappers in src/mcp/tools.rs to call record_tool_event on completion or failure.
- Record tool name, session_id, strategy, result_count, latency_ms.
- For operations that pass through the protocol lane router (from Phase 02-02), also record the selected lane (cli/mcp/progressive_mcp) and the decision reason string. This completes the PRO-03 audit trail that Phase 02-02 deferred.
- Guard telemetry writes so operational failures degrade gracefully.

Verify:

- automated: cargo test test_tool_events -- --nocapture

Done:

- MCP tool calls produce telemetry rows consistently.

### Task 2: Expose session summary surface

Files: src/mcp/resources.rs, src/cli/commands/mod.rs, tests/test_tool_events.rs

Type: auto

TDD: true

Behavior:

- Session summary resource returns total_calls, total_latency_ms, avg_groundedness, and per-tool breakdown.
- Empty session (no events) returns a valid zero-value summary, not an error.
- CLI `hief session-cost` command prints the same summary in human-readable form.

Action:

- Add session summary read API to existing resources and optionally a CLI readout command for recent session stats.
- Ensure outputs include total_calls, total_latency_ms, avg_groundedness, and tool breakdown where available.
- Validate output schema and empty-state behavior in tests.

Verify:

- automated: cargo test test_tool_events -- --nocapture

Done:

- Users can retrieve per-session observability snapshot without direct SQL access.

## Threat Model

Trust boundaries:

- instrumentation wrapper to core handler: Telemetry failures must not degrade tool correctness.
- session summary output to user: Aggregates must avoid leaking sensitive raw queries by default.

STRIDE register:

- T-03-03, Denial of service, telemetry write path, mitigate: Best-effort writes with bounded payload and non-fatal failure handling.
- T-03-04, Information disclosure, session summary output, mitigate: Expose aggregates by default, raw query detail only in explicit debug mode.
- T-03-SC, Tampering, cargo dependency changes, accept: No package-manager installs in this plan.
